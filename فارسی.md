# UPJET SSH Panel

UPJET SSH Panel یک پنل ساده و سبک برای مدیریت کاربران SSH روی سرورهای Linux است. این پنل برای ساخت کاربران SSH با محدودیت زمان، محدودیت حجم مصرفی، مدیریت پسورد، قفل/فعال‌سازی کاربر و محدود کردن اتصال همزمان طراحی شده است.

> این پروژه برای مدیریت سرور شخصی یا سرورهایی است که مالک یا مدیر مجاز آن هستید.

---

## قابلیت‌ها

* ساخت کاربر SSH از طریق پنل وب
* تعیین اعتبار زمانی کاربر، مثلاً ۳۰ روز
* تعیین حجم مصرفی، مثلاً ۱۰۰ گیگابایت
* نمایش مصرف هر کاربر
* نمایش پسورد ذخیره‌شده برای کاربران ساخته‌شده توسط پنل
* تغییر پسورد دستی هر کاربر
* ساخت پسورد تصادفی جدید
* ویرایش حجم کاربر
* ویرایش اعتبار کاربر
* قفل کردن کاربر
* فعال‌سازی دوباره کاربر
* ریست حجم مصرفی
* قطع اتصال فعال کاربر
* حذف کامل کاربر
* محدودیت یک اتصال همزمان برای هر کاربر
* فعال‌سازی ورود SSH با پسورد
* رفع تداخل با سرویس‌های قدیمی پنل
* رابط کارت‌بندی‌شده و مناسب موبایل و دسکتاپ

## نصب سریع

ابتدا فایل نصب را روی سرور قرار دهید:

```bash
nano install_upjet_ssh_panel_final_fixed.sh
```

محتوای اسکریپت را داخل فایل قرار دهید و ذخیره کنید.

سپس اجرا کنید:

```bash
sudo bash install_upjet_ssh_panel_final_fixed.sh
```

بعد از نصب، اطلاعات ورود پنل در این فایل ذخیره می‌شود:

```bash
sudo cat /root/upjet-panel-login.txt
```

---

## نصب با یوزر، پسورد و پورت دلخواه

می‌توانید هنگام نصب، یوزر و پسورد پنل را مشخص کنید:

```bash
sudo env PANEL_ADMIN_USER=admin PANEL_PASSWORD=StrongPassword123 PANEL_PORT=9080 PANEL_HOST=0.0.0.0 bash install_upjet_ssh_panel_final_fixed.sh
```

نمونه:

```bash
sudo env PANEL_ADMIN_USER=ahmadmute PANEL_PASSWORD=ahmad2405 PANEL_PORT=9080 PANEL_HOST=0.0.0.0 bash install_upjet_ssh_panel_final_fixed.sh
```

## ورود به پنل

اگر پنل روی IP عمومی فعال شده باشد:

```text
http://SERVER_IP:9080
```

مثال:

```text
http://104.253.43.244:9080
```

اطلاعات ورود:

```bash
sudo cat /root/upjet-panel-login.txt
```

---

## روش امن‌تر برای باز کردن پنل

بهتر است پنل فقط روی `127.0.0.1` اجرا شود و با SSH Tunnel باز شود.

اگر پنل روی این حالت تنظیم شده باشد:

```bash
PANEL_HOST=127.0.0.1
```

از سیستم خودتان بزنید:

```bash
ssh -L 9080:127.0.0.1:9080 root@SERVER_IP
```

بعد در مرورگر باز کنید:

```text
http://127.0.0.1:9080
```

## اتصال کاربران SSH

کاربران ساخته‌شده داخل پنل از طریق پورت SSH وصل می‌شوند، نه پورت پنل.

اگر SSH روی پورت پیش‌فرض ۲۲ باشد:

```bash
ssh username@SERVER_IP
```

اگر SSH روی پورت دیگری باشد، مثلاً `2222`:

```bash
ssh -p 2222 username@SERVER_IP
```

پورت `9080` فقط برای پنل وب است.

## تغییر یوزر و پسورد پنل

فایل تنظیمات پنل:

```bash
sudo nano /etc/upjet-ssh-panel.env
```

نمونه تنظیمات:

```bash
PANEL_ADMIN_USER=admin
PANEL_PASSWORD=StrongPassword123
PANEL_SECRET=random_secret
PANEL_HOST=0.0.0.0
PANEL_PORT=9080
```

بعد از تغییر:

```bash
sudo systemctl restart upjet-ssh-panel
```

---

## تغییر پورت پنل

فایل تنظیمات را باز کنید:

```bash
sudo nano /etc/upjet-ssh-panel.env
```

این مقدار را تغییر دهید:

```bash
PANEL_PORT=9080
```

مثلاً:

```bash
PANEL_PORT=9090
```

بعد:

```bash
sudo systemctl restart upjet-ssh-panel
```

اگر فایروال فعال است:

```bash
sudo ufw allow 9090/tcp
sudo ufw reload
```

## مدیریت سرویس

وضعیت پنل:

```bash
sudo systemctl status upjet-ssh-panel --no-pager
```

ری‌استارت پنل:

```bash
sudo systemctl restart upjet-ssh-panel
```

توقف پنل:

```bash
sudo systemctl stop upjet-ssh-panel
```

فعال‌سازی اجرای خودکار بعد از ریبوت:

```bash
sudo systemctl enable upjet-ssh-panel
```

دیدن لاگ پنل:

```bash
sudo journalctl -u upjet-ssh-panel -n 100 --no-pager
```

لاگ زنده:

```bash
sudo journalctl -u upjet-ssh-panel -f
```

## مانیتور حجم مصرفی

اسکریپت مانیتور حجم به‌صورت خودکار هر ۵ دقیقه اجرا می‌شود.

مسیر اسکریپت:

```bash
/opt/upjet-ssh-panel/monitor_quota.sh
```

اجرای دستی:

```bash
sudo /opt/upjet-ssh-panel/monitor_quota.sh
```

لاگ قفل شدن کاربران بعد از اتمام حجم:

```bash
sudo cat /var/lib/upjet-ssh-panel/quota.log
```

## اطلاعات کاربران

اطلاعات حجم، وضعیت و پسوردهای ذخیره‌شده در مسیر زیر نگهداری می‌شود:

```bash
/var/lib/upjet-ssh-panel
```

فایل‌های نمونه برای هر کاربر:

```text
username.quota
username.total
username.last
username.status
username.password
```

نکته مهم:

پسوردهای قدیمی کاربران Linux قابل بازیابی نیستند، چون Linux پسورد واقعی را ذخیره نمی‌کند و فقط hash آن را نگه می‌دارد. پنل فقط پسوردهایی را نشان می‌دهد که خودش ساخته یا از داخل پنل تغییر داده شده‌اند.

## محدودیت یک اتصال همزمان

پنل برای کاربران SSH گروه زیر را می‌سازد:

```text
sshclients
```

و در فایل limits این محدودیت را اضافه می‌کند:

```text
@sshclients hard maxlogins 1
```

همچنین در SSH مقدار زیر تنظیم می‌شود:

```text
MaxSessions 1
```

برای بررسی:

```bash
grep sshclients /etc/security/limits.conf
grep MaxSessions /etc/ssh/sshd_config
```

اگر کاربری عضو گروه نبود:

```bash
sudo usermod -aG sshclients username
```

---

## فعال بودن ورود با پسورد SSH

اسکریپت به‌صورت خودکار این تنظیمات را برای SSH اعمال می‌کند:

```text
PasswordAuthentication yes
KbdInteractiveAuthentication yes
UsePAM yes
PermitEmptyPasswords no
```

برای بررسی:

```bash
sudo grep -RniE 'PasswordAuthentication|KbdInteractiveAuthentication|UsePAM|PermitEmptyPasswords' /etc/ssh/sshd_config /etc/ssh/sshd_config.d/*.conf 2>/dev/null
```

بعد از تغییرات دستی SSH:

```bash
sudo sshd -t
sudo systemctl restart ssh 2>/dev/null || sudo systemctl restart sshd
```

## عیب‌یابی اتصال SSH کاربران

اگر کاربر ساخته می‌شود ولی نمی‌تواند وصل شود، این موارد را بررسی کنید.

بررسی کاربر:

```bash
id username
sudo passwd -S username
sudo chage -l username
grep username /etc/passwd
```

فعال کردن کاربر:

```bash
sudo usermod -U username
sudo chage -E $(date -d "+30 days" +%F) username
```

ریست پسورد کاربر:

```bash
sudo passwd username
```

بررسی پورت SSH:

```bash
sudo ss -ltnp | grep ssh
```

تست از داخل سرور:

```bash
ssh username@127.0.0.1
```

لاگ SSH:

```bash
sudo journalctl -u ssh -n 80 --no-pager
```

اگر سرویس SSH با نام `sshd` بود:

```bash
sudo journalctl -u sshd -n 80 --no-pager
```

## رفع تداخل با پنل قبلی

اگر قبلاً سرویس `ssh-panel` نصب بوده، ممکن است پورت را اشغال کند.

خاموش کردن پنل قبلی:

```bash
sudo systemctl disable --now ssh-panel 2>/dev/null
sudo pkill -f "/opt/ssh-panel/app.py" 2>/dev/null
```

ری‌استارت پنل جدید:

```bash
sudo systemctl restart upjet-ssh-panel
```

بررسی پورت:

```bash
sudo ss -ltnp | grep -E '9080|python'
```

## حذف پنل

برای حذف سرویس پنل:

```bash
sudo systemctl disable --now upjet-ssh-panel
sudo rm -f /etc/systemd/system/upjet-ssh-panel.service
sudo systemctl daemon-reload
```

حذف فایل‌های برنامه:

```bash
sudo rm -rf /opt/upjet-ssh-panel
```

حذف داده‌های پنل:

```bash
sudo rm -rf /var/lib/upjet-ssh-panel
```

حذف فایل تنظیمات:

```bash
sudo rm -f /etc/upjet-ssh-panel.env
sudo rm -f /root/upjet-panel-login.txt
```

نکته: حذف پنل به‌صورت خودکار کاربران SSH ساخته‌شده را حذف نمی‌کند. برای حذف کاربران باید جداگانه اقدام کنید.

## نکات امنیتی

* پنل مدیریت SSH را بدون پسورد قوی روی اینترنت عمومی قرار ندهید.
* بهتر است `PANEL_HOST=127.0.0.1` باشد و پنل فقط با SSH Tunnel باز شود.
* اگر پنل را عمومی می‌کنید، حداقل پورت را با فایروال فقط برای IP خودتان باز کنید.
* پسورد پنل را ساده نگذارید.
* فایل `/etc/upjet-ssh-panel.env` و مسیر `/var/lib/upjet-ssh-panel` باید فقط برای root قابل خواندن باشند.
* این پنل برای مدیریت سبک و ساده کاربران SSH مناسب است، نه فروش سنگین یا زیرساخت بزرگ.

## نمونه تنظیم فایروال

باز کردن پورت پنل:

```bash
sudo ufw allow 9080/tcp
sudo ufw reload
```

باز کردن پورت SSH پیش‌فرض:

```bash
sudo ufw allow 22/tcp
sudo ufw reload
```

مشاهده وضعیت فایروال:

```bash
sudo ufw status
```
